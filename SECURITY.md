# Security Policy

## Supported Versions

The following versions of TradeLab are currently being supported with security updates:

| Version | Supported |
| ------- | --------- |
| v1.0.x  | Yes       |
| < v1.0  | No        |

## Reporting a Vulnerability

We take the security of TradeLab seriously. If you discover a security vulnerability within this project, please follow the steps below to report it responsibly:

### How to report
Please **do not** open a public GitHub issue for security vulnerabilities. Instead, send an email to the project maintainers (**supreethn1709@gmail.com** or **pranavarun19@gmail.com**) with the following details:

- **Type of issue** (e.g., SQL injection, XSS, insecure dependency)
- **Detailed steps to reproduce** the vulnerability
- **Potential impact** of the vulnerability
- **Proposed fix** (if you have one)

### Our Commitment
Once a report is received, we will:
1.  Acknowledge receipt of your report within **48 hours**.
2.  Keep you informed of our progress as we investigate and work on a fix.
3.  Publicly announce the vulnerability and credit the reporter (if desired) once a fix has been released and deployed.

Thank you for helping keep TradeLab secure!
